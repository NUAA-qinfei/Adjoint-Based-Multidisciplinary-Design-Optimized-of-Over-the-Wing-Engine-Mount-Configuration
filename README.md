  我对翼上发动机布局的研究萌芽于2015年，那一年我上高二，当时已经特别喜欢飞机也已经想好未来去从事与飞机和飞行相关的工作。那一年，我偶然从《福布斯》杂志上看到了对本田Hondajet公务机的设计师藤野道格的专访，当时我才知道还有这架飞机的存在，彼时Hondajet飞机已经拿到了FAA的适航进入了公务机市场，当时我立刻被这架飞机优美的外形以及独特的发动机摆放位置所吸引。当时的印象是藤野道格说预期试图去尽量减少机翼和发动机周围气流之间的干扰，为什么不把这两股气流结合起来以找到理想的整体空气动力学设计呢？而且他还说了发动机在翼上的摆放位置极其关键，他们当时主要是通过风洞试验确定的。但当时我不懂是为什么，但这个飞机一直萦绕在我脑海里，以至于我本科航空英语的课堂上还和大家分享这架飞机，以及我硕士入学时就送了导师一架Hondajet的模型。
  
  后来我了解到这种布局不是本田原创,而是借鉴了德国的VFW-614飞机，也了解到本田所谓的有利干扰其实是Ma0.72飞行时发动机进气口前方的高压流场可以辅助降低机翼上表面气流减速从而降低激波强度以及提升阻力发散特性并且确实这个位置极其关键，并且与机翼设计的耦合性也更强，他们当年主要是试验出来的。后来我也了解到美国与德国一些研究机构也在研究这种布局能否用在民航客机上，而此时我在南航跟着导师做的研究方向主要是气动外形伴随优化，于是就想着能否用伴随这种新的方法研究此类布局的设计问题，彼时已经是2024年,距离我了解到这个飞机已经将近10年了。
  
  本研究的完整耦合脚本、FFD参数化源文件、CFD网格与结构网格文件、流场求解与结构有限元求解文件已公布在此处。其中0文件夹是流场边界条件设置文件，FFD文件夹是使用的参数化FFD文件，system是DAFoam流场求解设置文件，wingbox是结构有限元网格，气动网格与热力学参数设置网格在master分支下的constant文件夹中，runScript_OWEM.py是完整的多学科耦合脚本文件，tacsSetup.py是TACS结构有限元求解的设置脚本文件，结果展示文件夹中放了一些优化前后的对比图片。需要说明的是runScript_OWEM.py与我论文中的算例有一定的区别，主要是目标升力系数的值与固定迎角的设置有区别(我论文中放开了迎角约束)。
  
  需要补充说明的是，对于工程实际中的一些问题可能因为网格原因伴随残差降不下去，一个重要的经验是调节PETSc预条件器的强度，DAFoam中主要通过pcFillLevel控制ILU预条件器的fill-in level,以及通过asmOverlap控制ASM预条件器的重叠层数。本脚本中这两个参数我都设置成了3来保证GMRES能收敛,但这样会消耗巨大内存。
  此外我这个研究使用的DAFoam版本号是V3.1.3。
